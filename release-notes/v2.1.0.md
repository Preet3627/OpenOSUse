## v2.1.0 — Dual Models, Native AX Clicking, Touch ID, and Optional Screenshots

This release adds automatic model fetching, separate chat/vision model selection, native Accessibility-based element clicking, Touch ID biometric verification, and the ability to disable screenshots entirely.

### 🔀 Dual Model Selection
- Separate **Chat Model** and **Vision Model** dropdowns in Settings
- Auto-fetch available models from provider API (Ollama, Gemini, Anthropic, Groq, Grok)
- Vision model list filtered to known vision-capable models (llava, pixtral, vision, etc.)
- Refresh button to re-fetch models manually
- Text field fallback when model fetch fails or is unavailable

### 🖱️ Native AX Element Clicking (`click_element`)
- New `click_element(label, role?)` tool — finds UI elements by their on-screen label using the macOS Accessibility API
- Eliminates coordinate guesswork: works at any resolution, window size, or Retina scaling
- Preferred over `click(x, y)` — system prompt updated to encourage AX-based targeting
- Recursive depth-first search of the AX element tree (title, description, value matching)

### 🔐 Touch ID Biometric Verification
- Generalized permission system: Clicks, Screenshots, AX Tree reads, and App Launches each require Touch ID
- Each scope authorizes **once per session** — subsequent uses skip the prompt
- Individual toggles in Settings to enable/disable Touch ID per action type
- Customizable in Settings → Touch ID Permissions

### 📸 Optional Screenshots & Vision
- **Screenshots toggle** — disable screen capture entirely (no Screen Recording permission needed)
- **Vision Model toggle** — disable separate vision model (chat model gets text context only)
- Disabling screenshots automatically disables the vision model toggle
- Conditional pipeline: screenshot+vision two-step → screenshot direct → text-only based on toggles
