## v0.2.0 — Settings Persistence, Retry Logic & System Notifications

This release focuses on stability, configurability, and user experience improvements:

### ⚙️ Settings Persistence (AgentSettings)
New Settings tab in the UI with persistent configuration across launches:
- Provider/model picker with dropdown
- Server URL text field
- Cooldown, max retries, and request timeout sliders/steppers
- Accessibility Tree and System Notifications toggles
- Apply Settings and Export Telemetry buttons
- All values stored in UserDefaults

### 🔁 Network Retry & Timeout
- Automatic retry (up to 3 attempts) when the agent server is unreachable
- Configurable request timeout (default 30s)
- Graceful failure when all retries are exhausted

### 🔔 System Notifications
- macOS notifications when the agent finishes or encounters an error
- Notifications show step count and objective summary
- Configurable via Settings toggle

### 📤 Telemetry Export
- Export full telemetry logs as JSON via the standard save dialog
- Timestamps, states, steps, and messages included

### Other
- `AgentSettings.applyToOrchestrator()` syncs settings to the running loop
- `objective` published property for notification context
- `Codable` conformance on TelemetryEntry and AgentState
- Code quality: removed unused Double.nonzero extension, fixed @MainActor isolation
